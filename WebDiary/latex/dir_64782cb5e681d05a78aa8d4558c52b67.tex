{\itshape \href{https://github.com/systemjs/systemjs/tree/2.0}{\texttt{ Try out the System\+JS 2.\+0 alpha release}}}

{\itshape System\+JS is \href{https://canopytax.github.io/post/systemjs-sponsorship/?utm_source=systemjs}{\texttt{ currently sponsored by Canopy Tax}}.}


\begin{DoxyItemize}
\item Loads any module format when running the \texorpdfstring{$\sim$}{\string~}15\+KB development build.
\item Loads ES modules compiled into the {\ttfamily System.\+register} module format for production with \href{https://github.com/ModuleLoader/es6-module-loader/blob/v0.17.0/docs/circular-references-bindings.md}{\texttt{ exact circular reference and binding support}}
\item Supports Require\+JS-\/style \href{docs/getting-started.md\#map-config}{\texttt{ map}}, \href{https://github.com/systemjs/systemjs/blob/master/docs/config-api.md\#paths}{\texttt{ paths}}, and \href{docs/production-workflows.md\#bundle-extension}{\texttt{ bundles}} configuration.
\end{DoxyItemize}

Built with the \href{https://github.com/ModuleLoader/es-module-loader}{\texttt{ ES Module Loader project}}, which is based on principles and APIs from the What\+WG Loader specification, modules in HTML and Node\+JS.
